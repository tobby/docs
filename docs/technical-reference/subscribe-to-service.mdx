---
title: "Subscribe to Service"
description: "Subscribe to Dojah webhooks to receive real-time notifications for
 events such as KYC verifications, OTP status updates, AML alerts, and wallet activity."
---


This endpoint allows you to Subscribe to Dojah's webhook service, alternatively via the [dashboard](/guides/reporting/developers/configuration/webhook) 

### Request

```http [POST]
{{baseUrl}}api/v1/webhook/subscribe
```

### Headers

- `AppId`: Create an app to get your app ID on dashboard [here](https://app.dojah.io/developers/configuration)
- `Authorization`: Your private secret Key


### Body Parameters

| Parameter | Type   | Description | Required |
|----------|--------|-------------|----------|
| `webhook` | string | Your callback URL where Dojah will POST events (e.g., `https://yourapp.com/webhook`) | Yes |
| `service` | string | Type of service to subscribe to (e.g. `sms`, `kyc_widget`, `address`, `AML Monitoring`, `ngn_wallet`) | Yes |


### Sample Response

```json Response
{
 "entity": "Webhook added successfully"
}
```


### Verification Statuses
These are the possible verification statuses: 
- `Ongoing`
- `Abandoned`
- `Completed` 
- `Pending`
- `Failed`
